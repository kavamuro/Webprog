Kombinátor kijelölők

A háttértárolóink könyvtárszerkezetéhez hasonló módon a HTML oldalakat felépítő elemek kapcsolatát is faszerkezetben szokás ábrázolni. A kombinátorokkal a böngésző egy-egy elemnek a fa szerkezetben (dokumentumfán) elfoglalt helye ismeretében határozza meg miként kell az egyes elemeknek megjelenítenniük. Az adott elem tulajdonságainak meghatározásánál a webfejlesztőnek is ismerni kell az elemek dokumentumfán elfoglalt helyzetét. Ennek alapján lehet például csak egy meghatározott elem belsejében lévő (a dokumentumfán alatta elhelyezkedő) elemeknek egy adott tulajdonságát beállítani.

További információk: w3schools CSS Combinators

HTML gyökérelem

Minden HTML5 dokumentum gyökér eleme, egyben az összes többi elem őse a html elem. Tudjuk, hogy a gyökér elem nyitó és záró címkéi közé egy head és egy body elemet ágyazunk be. Ezek az ő gyermekei. A html elem pedig a head és a body szülő eleme. A dokumentumfejbe (head) ágyazott további elemek nem kerülnek megjelenítésre a weboldal tartalmaként, így ezekhez nem tartoznak stílusdefiníciók. A dokumentumtörzsben (body) található a megjelenítendő tartalom, így a továbbiakban a body elembe ágyazott elemek stílusjellemzőivel foglalkozunk.

Leszármazott kijelölő (szóköz)

Egy HTML5 dokumentumban minden elem a html elem leszármazottja, mivel minden elem a <html>..</html> címkék között helyezkedik el. A html elem hierarchiába legfelső szintjén helyezkedik el, minden más elem alatta van. A html elem, így minden más elem őse, a többiek pedig az ő leszármazottja. A hierarchia következő szintjén a body elem található. A <body> és </body> címkék által közrefogott elemek a body elem leszármazottjai, amelyekbe további elemek ágyazhatók. Ezek mind a body leszármazottjai. Általánosságban elmondható, hogy minden elem leszármazottja minden a hierarchiában felette álló elemnek. (Az ősök nyitó és záró címkéi között helyezkedik el.)

A leszármazott kijelölővel az adott elem összes leszármazottját, vagyis a fában az adott ágon található elemet kijelölhetjük. Jele a szóköz.

A leszármazott kijelölő szintaxisa: szelektor1 szelektor2 { meghatározásblokk }, ahol a szelektor1 a szülőelem, a szelektor2 pedig a szelektor1 valamilyen szintű leszármazottja.

Példa: a div elemek összes span leszármazottja: div span { ... }

Gyermek kijelölő ( > )

Ha a dokumentumfán egymás alatt helyezkedik el két elem, akkor a hierarchiában feljebb lévő a szülő eleme a alatta elhelyezkedőnek. Ha egy páros címkével jelölt HTML elem nyitó és záró címkéi közé helyezünk (beágyazunk) egy másik HTML elemet, akkor a beágyazott elem az őt befogadó elem gyermeke lesz. Body gyermeke minden olyan elem, amely <body>..</body> címkék között található, de nincs beágyazva más elembe.

A gyermek kijelölővel az adott elem összes elsőszintű leszármazottját, azaz gyermekét, vagyis a fában az adott ágon található első szint elemeit kijelölhetjük. Jele a >.

A gyermek kijelölő szintaxisa: szelektor1 > szelektor2 { meghatározásblokk }, ahol a szelektor1 a szülőelem, a szelektor2 pedig a szelektor1 elem gyermeke.

Példa: a div elemek összes p gyermeke: div > p { ... }

Közvetlen testvér kijelölő ( + )

A szomszédos testvér választóval olyan elemet lehet kiválasztani, amelyik közvetlenül egy másik konkrét elem után következik és a két elem egymásnak testvére. A testvérelemeknek tehát ugyanannak a szülőnek a gyermekei, a szomszédos pedig azt jelenti, hogy a két elem között más nem lehet, tehát közvetlenül követik egymást. A két elemnek nem kell azonos típusúnak lennie!

A közvetlen testvér kijelölő szintaxisa: szelektor1 + szelektor2 { meghatározásblokk }, ahol a szelektor1 elemet közvetlenül követi a szelektor2-vel megjelölt elem

Példa: a képeket követő bekezdések: img + p { ... }

Általános testvér ( ~ )

Az általános testvér választó minden olyan elemet kiválaszt, amely egy adott elem testvére, bárhol is legyen.Az általános testvérelemeknek tehát ugyanannak a szülőnek a gyermekei, az általános pedig azt jelenti, hogy a két elem között más elemek (akár újabb testvérek is) lehetnek, tehát nem kell közvetlenül követniük egymást. A két elemnek nem kell azonos típusúnak lennie!

Az általános testvér kijelölő szintaxisa: szelektor1 ~ szelektor2 { meghatározásblokk }, ahol a szelektor1 elemet követi a szelektor2-vel megjelölt elem

Példa: div és p testvérek formázása: div ~ p { ... }
